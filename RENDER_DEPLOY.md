# Deploying 24PEX USERBOT to Render

## Prerequisites

- A [Render](https://render.com) account (free tier works)
- Your Telegram API ID and API Hash from [https://my.telegram.org/apps](https://my.telegram.org/apps)
- A `.session` file generated by running `python bot.py` locally at least once

## Step 1 – Upload Your Session File

Because Render is stateless, you need to convert your `.session` file to a base64 string and store it as an environment variable.

Run this on your local machine after generating the session:

```bash
base64 -w 0 24pex_userbot.session
```

Copy the output — you'll paste it as `SESSION_STRING` in Render.

Then add this to the top of `bot.py` (before `TelegramClient(...)` is called):

```python
import base64, os
session_b64 = os.environ.get("SESSION_STRING")
if session_b64:
    with open(f"{SESSION_NAME}.session", "wb") as f:
        f.write(base64.b64decode(session_b64))
```

_(This is already handled in the `bot.py` in this repo.)_

## Step 2 – Push to GitHub

Make sure your code is pushed to `https://github.com/bigmoneypro047/24PEX-USERBOT`.

## Step 3 – Create a Render Web Service

1. Go to [https://dashboard.render.com](https://dashboard.render.com)
2. Click **New → Web Service**
3. Connect your GitHub account and select **24PEX-USERBOT**
4. Configure:
   - **Name**: `24pex-userbot`
   - **Region**: Choose closest to Nigeria (e.g. Frankfurt or Oregon)
   - **Branch**: `main`
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`
   - **Instance Type**: Free (or Starter for always-on)

## Step 4 – Set Environment Variables

In Render → your service → **Environment**, add:

| Key | Value |
|---|---|
| `TELEGRAM_API_ID` | Your API ID from my.telegram.org |
| `TELEGRAM_API_HASH` | Your API Hash from my.telegram.org |
| `SESSION_NAME` | `24pex_userbot` |
| `TELEGRAM_GROUP_IDS` | `@group1,@group2,@group3` |
| `SESSION_STRING` | _(base64 string from Step 1)_ |

## Step 5 – Deploy

Click **Create Web Service**. Render will build and start your bot automatically.

## Keeping the Bot Alive (Free Tier)

Render's free tier spins down after 15 minutes of inactivity. To prevent this, use [UptimeRobot](https://uptimerobot.com) to ping your Render URL every 10 minutes.

1. Sign up at UptimeRobot (free)
2. Add a new monitor: **HTTP(S)** type
3. URL: your Render service URL (e.g. `https://24pex-userbot.onrender.com`)
4. Interval: every 5 minutes

## Troubleshooting

- **"Session file not found"**: Make sure `SESSION_STRING` env var is set correctly
- **"FLOOD_WAIT"**: Telegram rate limit hit. The bot will retry automatically
- **Messages not sending**: Check group IDs are correct. Use `@username` for public groups or numeric ID for private groups
